# 数据清洗

## 干什么
让 OpenClaw 自动清洗和处理数据，包括缺失值处理、异常值检测、格式标准化、数据去重等。

## 怎么干

### 1. 准备工作
- 需要 OpenClaw 版本：>= 1.0.0
- 需要的技能：无
- 数据格式：CSV、JSON、Excel

### 2. 执行步骤

**方式一：缺失值处理**

```
清洗以下数据：

数据：
姓名,年龄,城市,收入
张三,28,北京,15000
李四,,上海,18000
王五,35,,20000
赵六,42,广州,
陈七,,深圳,12000

要求：
1. 检测所有缺失值
2. 年龄：用平均值填充
3. 城市：标记为"未知"
4. 收入：用中位数填充
5. 输出清洗报告
```

**方式二：异常值检测与处理**

```
检测并处理异常值：

数据：用户消费记录
字段：user_id, amount, date

异常规则：
- amount < 0：标记为退款
- amount > 100000：标记为大额，需人工审核
- 3σ 原则检测异常值

处理方式：
- 生成异常报告
- 提供处理建议
- 可选：删除、替换、标记
```

**方式三：数据格式标准化**

```
标准化数据格式：

原始数据：
- 日期：2024/03/24, 2024-03-24, 03/24/2024, 240324
- 电话：138-1234-5678, 13812345678, +86 138 1234 5678
- 金额：¥15,000, 15000元, 15000.00

统一格式：
- 日期：YYYY-MM-DD
- 电话：138-XXXX-XXXX
- 金额：数字（去除符号）
```

**方式四：数据去重**

```
数据去重处理：

数据：客户信息表
字段：customer_id, name, email, phone

去重规则：
1. 主键：customer_id
2. 辅助判断：email、phone
3. 冲突处理：
   - 相同 ID：保留最新记录
   - 相同邮箱：合并信息
   - 相同电话：人工审核

输出：
- 原始记录数
- 去重后记录数
- 重复记录详情
```

**方式五：数据验证**

```
验证数据质量：

数据：订单数据
验证规则：
1. 订单 ID：唯一且不为空
2. 用户 ID：存在于用户表中
3. 金额：正数且合理范围
4. 日期：不晚于当前日期
5. 状态：限定枚举值

生成：
- 验证通过率
- 错误记录列表
- 修复建议
```

**方式六：批量数据清洗**

```
批量清洗流程：

数据源：/data/raw/*.csv
清洗步骤：
1. 统一编码格式（UTF-8）
2. 去除 BOM 标记
3. 标准化列名（小写+下划线）
4. 处理缺失值
5. 去除重复行
6. 验证数据类型
7. 输出到：/data/cleaned/

并行处理：启用
日志记录：详细模式
```

**OpenClaw 会自动：**
1. 扫描数据质量问题
2. 应用清洗规则
3. 生成清洗报告
4. 提供处理建议
5. 输出干净数据
