# 长文写作

## 真实来源

- **工具**：OpenClaw 对话生成
- **辅助**：gemini 技能、飞书文档
- **场景**：技术文章、分析报告、深度内容

---

## 干什么
让 OpenClaw 帮你撰写长篇技术文章、深度分析、研究报告，支持大纲生成、分段写作。

## 怎么干

### 1. 准备工作

- OpenClaw 版本：>= 1.0.0
- 可选：gemini 技能、飞书文档

### 2. 执行步骤

**方式一：从大纲开始**

```
写一篇关于「微服务架构最佳实践」的技术文章：

要求：
- 字数：5000-8000 字
- 深度：中级开发者可理解
- 包含：代码示例、架构图说明
- 风格：技术博客

先生成大纲给我确认，再分段写作。
```

**方式二：深度研究分析**

```
撰写一份关于「2024 年 AI 编程助手发展趋势」的分析报告：

章节：
1. 市场现状与规模
2. 主要玩家对比分析
3. 技术演进路线
4. 用户采用情况
5. 未来趋势预测

要求：
- 数据支撑
- 图表描述
- 约 10000 字
```

**方式三：分段写作**

```
我需要写一篇长文，分 5 个部分：

主题：React 性能优化完全指南

第一部分：性能问题诊断
第二部分：渲染优化
第三部分：状态管理优化
第四部分：网络请求优化
第五部分：实战案例分析

先帮我写第一部分，约 1500 字。
```

**方式四：改写与润色**

```
优化以下文章：

原文：
[paste your article]

改进方向：
- 提升可读性
- 增加数据支撑
- 优化段落过渡
- 加强逻辑结构
```

### 3. 文章结构模板

**技术文章**：
```markdown
# 标题

## 引言
- 背景
- 问题
- 本文目标

## 正文
### 概念解释
### 原理分析
### 实现方案
### 代码示例
### 最佳实践

## 总结
- 核心要点
- 注意事项
- 参考资料
```

**分析报告**：
```markdown
# 报告标题

## 摘要
- 核心结论（200 字内）

## 背景
- 研究背景
- 研究目的

## 方法
- 数据来源
- 分析方法

## 发现
### 发现 1
### 发现 2
### 发现 3

## 建议
- 行动建议 1
- 行动建议 2

## 附录
- 数据表格
- 参考资料
```
