# 长文写作

## 干什么
让 OpenClaw 帮你撰写长篇技术文章、深度分析、研究报告，支持大纲生成、分段写作、质量优化。

## 怎么干

### 1. 准备工作
- 需要 OpenClaw 版本：>= 1.0.0
- 需要的技能：无
- 需要的配置：可选写作风格指南

### 2. 执行步骤

**方式一：从大纲开始**

```
写一篇关于"微服务架构最佳实践"的技术文章：

要求：
- 字数：5000-8000 字
- 深度：中级开发者可理解
- 包含：代码示例、架构图说明
- 风格：技术博客

先生成大纲给我确认，再分段写作。
```

**方式二：深度研究分析**

```
撰写一份关于"2024 年 AI 编程助手发展趋势"的分析报告：

包含以下章节：
1. 市场现状与规模
2. 主要玩家对比分析
3. 技术演进路线
4. 用户采用情况
5. 未来趋势预测

要求：
- 数据支撑
- 图表描述
- 专业术语解释
- 约 10000 字
```

**方式三：分段协作写作**

```
我需要写一篇长文，分 5 个部分：

主题：React 性能优化完全指南

第一部分：性能问题诊断
第二部分：渲染优化
第三部分：状态管理优化
第四部分：网络请求优化
第五部分：实战案例分析

先帮我写第一部分，约 1500 字。
```

**方式四：改写与润色**

```
优化以下文章：

原文：
[paste your long article]

改进方向：
- 提升可读性
- 增加数据支撑
- 优化段落过渡
- 加强逻辑结构
- 保持专业技术风格
```

**OpenClaw 会自动：**
1. 分析主题和目标受众
2. 生成结构化大纲
3. 分段撰写内容
4. 保持前后一致性
5. 优化表达和逻辑
6. 检查格式和规范

### 3. 效果展示

**生成的大纲：**
```markdown
# 微服务架构最佳实践

## 1. 引言（500 字）
- 微服务的定义与演进
- 本文的目标读者

## 2. 服务拆分策略（1500 字）
### 2.1 领域驱动设计（DDD）
### 2.2 拆分粒度权衡
### 2.3 案例分析

## 3. 通信机制（1500 字）
### 3.1 同步通信：REST vs gRPC
### 3.2 异步通信：消息队列
### 3.3 服务发现

## 4. 数据管理（1500 字）
### 4.1 数据库分离
### 4.2 分布式事务
### 4.3 数据一致性

## 5. 部署与运维（1000 字）
### 5.1 容器化与编排
### 5.2 监控与日志
### 5.3 灰度发布

## 6. 最佳实践总结（500 字）
- 核心要点回顾
- 常见陷阱提醒
```
