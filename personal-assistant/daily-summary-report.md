# 每日总结报告生成

## 真实来源

- **配置**：用户明确需求（2026-03-14）
- **内容**：OpenClaw 会话记录 + MEMORY.md
- **渠道**：飞书消息

---

## 干什么
每天 23:00 自动生成当日工作总结报告，发送到指定渠道，并写入长期记忆。

## 怎么干

### 1. 准备工作
- OpenClaw 版本：>= 1.0.0
- 配置：MEMORY.md 中记录每日任务
- 渠道：飞书/Slack/其他消息平台

### 2. 执行步骤

**方式一：手动触发**

```
现在帮我总结今天的工作：
1. 回顾今天完成的任务
2. 分析数据和进展
3. 记录遇到的问题
4. 规划明天的重点
```

**方式二：定时自动化**

配置 cron 任务，每天 23:00 自动触发：

```bash
# 在 OpenClaw 配置中添加
0 23 * * * openclaw cron --task daily-summary --channel feishu
```

**方式三：完整流程**

1. 读取当日会话记录
2. 提取关键任务和成果
3. 分析完成情况
4. 生成结构化报告
5. 发送到指定渠道
6. 写入长期记忆（MEMORY.md + memory/YYYY-MM-DD.md）

### 3. 报告结构

```markdown
# 每日总结 | YYYY-MM-DD

## 今日完成
- ✅ 完成的任务列表
- 关键成果

## 数据
- 代码提交：X 次
- PR 合并：Y 个
- 文档更新：Z 篇

## 遇到的问题
- 问题描述
- 解决方案

## 明日计划
1. 优先级排序的任务
2. 需要跟进的事项

## 长期记忆更新
- 新增决策
- 新增偏好
- 新增知识点
```

### 4. 效果展示

```
📊 每日总结报告已生成！

## 今日完成
- ✅ 完成案例补充（新增 3 个真实案例）
- ✅ CachePilot 托盘功能优化（进度 95%）
- ✅ 更新 MEMORY.md 任务看板

## 数据
- 代码提交：2 次
- 文档更新：4 篇
- 案例创建：3 个

## 进行中任务
- ai-coding-status-line 迭代治理（50%）
- CachePilot 托盘优化（95%）
- CachePilot 子目录扫描（20%）

## 明日计划
1. 完成 CachePilot 托盘测试
2. 继续 ai-coding-status-line 迭代
3. 检查是否有新阻塞

---
✅ 已写入长期记忆
✅ 已发送到飞书
```

## 需要准备什么

| 依赖项 | 说明 | 获取方式 |
|--------|------|----------|
| OpenClaw | >= 1.0.0 | 官方安装 |
| Cron | 定时任务 | 系统自带 |
| 消息渠道 | 接收报告 | 飞书/Slack 等 |

## 配置示例

在 MEMORY.md 中添加：

```markdown
## 用户偏好

- **每日总结**（2026-03-14）：固定每天 23:00 发送《每日总结与反思报告》，并写入长期记忆
```
