# 每日总结报告生成

## 干什么
每天 23:00 自动生成当日工作总结报告，发送到指定渠道。

## 怎么干

### 1. 准备工作
- 需要 OpenClaw 版本：>= 1.0.0
- 需要的技能：无特殊要求
- 需要的配置：cron 定时任务、消息渠道

### 2. 执行步骤

**方式一：直接指令**

```
现在帮我总结今天的工作：
1. 回顾今天完成的任务
2. 分析数据和进展
3. 记录遇到的问题
4. 规划明天的重点
```

**方式二：定时自动化**

配置 cron 任务，每天 23:00 自动触发：

```bash
# 在 OpenClaw 配置中添加
0 23 * * * openclaw cron --task daily-summary --channel feishu
```

**OpenClaw 会自动：**
1. 读取当日会话记录
2. 提取关键任务和成果
3. 分析完成情况
4. 生成结构化报告
5. 发送到指定渠道

### 3. 效果展示

```markdown
# 每日总结 | 2026-03-24

## 今日完成
- ✅ 完成 do-something-with-claw 案例补充（新增 12 个案例）
- ✅ 修复登录模块的 3 个 bug
- ✅ 编写 API 文档初稿

## 数据
- 代码提交：5 次
- PR 合并：2 个
- 文档更新：3 篇

## 遇到的问题
- CI 构建失败（已修复）
- 设计评审推迟到明天

## 明日计划
1. 完成剩余案例创建
2. 设计评审会议
3. 准备周报材料
```

## 需要准备什么

| 依赖项 | 说明 | 获取方式 |
|--------|------|----------|
| OpenClaw | >= 1.0.0 | 官方安装 |
| Cron | 定时任务 | 系统自带 |
| 消息渠道 | 接收报告 | 飞书/Slack 等 |
