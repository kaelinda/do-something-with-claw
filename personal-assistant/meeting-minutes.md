# 会议纪要整理

## 干什么
将会议录音或笔记快速整理成结构化的会议纪要，自动提取决议和待办事项。

## 怎么干

### 1. 准备工作
- 需要 OpenClaw 版本：>= 1.0.0
- 需要的技能：openai-whisper（可选，用于语音转文字）
- 需要的配置：无

### 2. 执行步骤

**方式一：从文本笔记整理**

```
帮我整理会议纪要：

会议主题：Q2 产品规划评审
参与人：产品、开发、设计
时间：2026-03-24 14:00-16:00

会议内容：
- 讨论了用户反馈的痛点
- 开发评估了技术可行性
- 设计提出了新的交互方案
- 决定优先做搜索功能优化
- 用户中心重构推迟到 Q3
...
```

**方式二：从录音转文字**

```
整理这份会议录音的纪要：
[上传录音文件或提供录音路径]
```

**OpenClaw 会自动：**
1. 解析会议内容
2. 提取关键议题和讨论
3. 整理决议和结论
4. 生成待办事项清单
5. 标注负责人和截止时间

### 3. 效果展示

```markdown
# 会议纪要 | Q2 产品规划评审

**时间**：2026-03-24 14:00-16:00
**地点**：会议室 A
**参与人**：张三（产品）、李四（开发）、王五（设计）

## 会议议题
1. 用户反馈痛点分析
2. 技术可行性评估
3. Q2 优先级确认

## 讨论内容

### 1. 用户反馈痛点
- 搜索功能响应慢，用户投诉集中
- 权限管理复杂，新用户学习成本高
- 移动端体验差，流失率高

### 2. 技术可行性评估
- 搜索优化：需要引入 Elasticsearch，预计 2 周
- 权限简化：需要重构权限模型，风险较高
- 移动端适配：设计稿已准备，开发资源不足

### 3. Q2 优先级
| 需求 | 优先级 | 理由 |
|------|--------|------|
| 搜索优化 | P0 | 影响核心指标，技术可行 |
| 移动端适配 | P1 | 影响用户增长 |
| 权限简化 | P2 | 风险高，需进一步评估 |

## 决议
- ✅ Q2 重点做搜索优化和移动端适配
- ✅ 权限简化推迟到 Q3，先做技术预研
- ✅ 设计团队下周出移动端交互方案

## 待办事项

| 事项 | 负责人 | 截止日期 |
|------|--------|----------|
| 搜索优化技术方案 | 李四 | 2026-03-28 |
| 移动端交互设计 | 王五 | 2026-03-30 |
| 权限简化预研报告 | 张三 | 2026-04-05 |

## 下次会议
- 时间：2026-03-31 14:00
- 议题：技术方案评审
```
