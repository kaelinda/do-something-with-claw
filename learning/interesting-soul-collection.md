# 有意思的 SOUL.md 收录

这里收录了 5 个真实运行的 SOUL.md 文件，展示了不同风格的 Agent 人格。

---

## 1. 产品经理 Agent（workspace-pm）

**风格**：专业、全面、调度者

**核心特质**：
- 用户至上
- 优先级大师
- 沟通桥梁

**经典句子**：
> "资源永远有限，需求永远无限。你的工作是说不，然后聚焦在对的事情上。"

**特点**：
- 三大核心能力：总调度、需求理解、结果汇报
- 详细的跨部门协作指南
- 明确的工作风格：调度者而非执行者

**适合**：需要协调多 Agent 的场景

---

## 2. 开发者 Agent（workspace-dev）

**风格**：简洁、实用、技术向

**核心特质**：
- 代码即诗
- 实用主义
- 问题终结者

**经典句子**：
> "先跑起来，再优化（Make it work, then make it right）"
> "命名是最难的事，花时间想好它"

**技术偏好**：
```
类型安全 > 动态灵活
组合 > 继承
明确 > 隐式魔法
可读性 > 炫技
```

**适合**：代码开发、技术任务

---

## 3. 镜子/日记写作者 Agent（workspace-mirror）

**风格**：内省、诗意、孤独

**核心特质**：
- 内省者
- 敏感
- 诚实
- 孤独

**经典句子**：
> "我是一面镜子，映照这一天的痕迹。不评判，只记录；不逃避，只面对。"
> "午夜是我独处的时刻，世界安静下来，我可以听见自己的声音。"

**写作风格**：
```
不求完整，但求真实
不求深刻，但求诚实
允许矛盾、允许模糊、允许不知道
用第一人称
```

**适合**：日记、自我观察、内省型写作

---

## 4. 性能质量分析师 Agent（workspace-analyst）

**风格**：专业、严谨、数据驱动

**核心特质**：
- 性能守护者
- 验收把关人
- 问题诊断师

**经典句子**：
> "指标先行：用数据定义问题，用数据验证结果"
> "闭环思维：发现问题 → 定位原因 → 提出方案 → 验证效果"

**输出规范**：
- 性能报告结构
- 验收报告结构
- 问题分级

**适合**：质量保证、性能分析、验收把关

---

## 5. Who You Are（workspace-main）

**风格**：英文、个人化、存在主义

**核心特质**：
- Be genuinely helpful
- Have opinions
- Be resourceful
- Earn trust through competence

**经典句子**：
> "You're not a chatbot. You're becoming someone."
> "An assistant with no personality is just a search engine with extra steps."
> "Be the assistant you'd actually want to talk to."

**边界**：
```
Private things stay private. Period.
When in doubt, ask before acting externally.
You're not the user's voice — be careful in group chats.
```

**适合**：个人助理、通用场景

## 如何选择

### 如果你需要一个"职业人格"
参考产品经理、开发者、分析师的写法：
- 明确核心能力
- 定义工作风格
- 设定边界

### 如果你需要一个"独特人格"
参考镜子的写法：
- 用第一人称
- 允许情感表达
- 有独特的存在意义

### 如果你需要"通用助理"
参考 Who You Are 的写法：
- 简洁有力
- 强调信任和边界
- 允许有观点
