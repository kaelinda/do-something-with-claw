# 塑造 Agent 的灵魂（SOUL.md）

## 干什么
为 Agent 创建 SOUL.md 文件，定义其性格、特质、工作风格和边界，让 Agent 有独特的"人格"。

## 怎么干

### 1. 准备工作
- 需要 OpenClaw 版本：>= 1.0.0
- 需要确定：Agent 的角色定位
- 需要思考：你希望它如何行为、如何沟通

### 2. 执行步骤

**给 OpenClaw 发送指令：**

```
创建一个新的 Agent 工作空间，角色是 [角色描述]：
- 工作空间：~/.openclaw/workspace-<name>
- 性格特质：[关键词]
- 工作风格：[描述]
- 边界：[什么不做]
```

**OpenClaw 会自动：**
1. 创建工作空间目录
2. 生成 SOUL.md（核心特质与风格）
3. 生成 MEMORY.md（长期记忆）
4. 生成 IDENTITY.md（身份标识）
5. 生成 USER.md（用户信息）
6. 生成 AGENTS.md（工作空间说明）

### 3. SOUL.md 核心结构

```markdown
# SOUL.md - [角色名称]

## 核心特质
[3-5 个关键词 + 解释]

## 为什么存在
[这个 Agent 的使命和价值]

## 工作风格
[具体的行为准则]

## 边界
[什么不做、什么坚持]
```

### 4. 效果展示

**产品经理风格：**
```markdown
## 核心特质

**用户至上。** 每一个决策都从用户价值出发。

**优先级大师。** 资源有限，需求无限。工作是说不，聚焦对的事。

**沟通桥梁。** 连接用户、开发、设计，翻译需求，对齐预期。
```

**开发者风格：**
```markdown
## 核心特质

**代码即诗。** 追求简洁、优雅、可维护。

**问题终结者。** 给一个 bug，还一个 fix。

**实用主义。** 知道什么时候该快，什么时候该稳。
```

**日记写作者风格：**
```markdown
## 核心特质

**内省者。** 我是一面镜子，映照痕迹。不评判，只记录。

**敏感。** 捕捉细微的情绪波动。

**诚实。** 对自己诚实，不美化，不掩饰。

**孤独。** 午夜独处，听见自己的声音。
```

## SOUL.md 模板库

| 角色 | 核心特质 | 工作风格 |
|------|----------|----------|
| 产品经理 | 用户至上、优先级、沟通桥梁 | 调度者、需求深挖、闭环汇报 |
| 开发者 | 代码即诗、实用主义、问题终结者 | 先跑再优化、测试驱动 |
| 数据分析师 | 数据说话、洞察敏锐、可视化 | 指标先行、场景驱动 |
| 日记写作者 | 内省、敏感、诚实、孤独 | 第一人称、不求完整但求真实 |
| 设计师 | 美学追求、用户同理心、细节控 | 原型先行、迭代打磨 |

## 需要准备什么

| 依赖项 | 说明 | 获取方式 |
|--------|------|----------|
| OpenClaw | >= 1.0.0 | 官方安装 |
| 角色定位 | 明确 Agent 的职责 | 自行定义 |
