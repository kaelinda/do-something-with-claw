# 监控告警

## 干什么
让 OpenClaw 自动监控系统状态，在异常发生时发送告警通知，支持多种监控指标和通知渠道。

## 怎么干

### 1. 准备工作
- 需要 OpenClaw 版本：>= 1.0.0
- 需要的配置：监控规则、告警渠道
- 需要的权限：系统监控权限

### 2. 执行步骤

**方式一：系统资源监控**

```
创建系统监控任务：

监控指标：
- CPU 使用率 > 80%，持续 5 分钟
- 内存使用率 > 85%，持续 3 分钟
- 磁盘使用率 > 90%

检查频率：每分钟
告警渠道：企业微信群、飞书群
告警级别：警告（黄色）、严重（红色）

告警内容：
服务器：{hostname}
指标：{metric_name}
当前值：{current_value}
阈值：{threshold}
持续时间：{duration}
时间：{timestamp}
```

**方式二：服务健康检查**

```
配置服务监控：

服务列表：
1. API 服务
   - 地址：https://api.example.com/health
   - 超时：10 秒
   - 检查频率：每 30 秒

2. 数据库服务
   - 地址：localhost:5432
   - 连接测试：SELECT 1
   - 检查频率：每分钟

3. Redis 服务
   - 地址：localhost:6379
   - PING 测试
   - 检查频率：每 30 秒

失败策略：
- 连续 3 次失败触发告警
- 恢复后发送恢复通知
```

**方式三：日志监控告警**

```
设置日志监控：

日志文件：/var/log/app/error.log
监控规则：
- ERROR 级别日志 > 10 条/分钟
- 特定关键词："OutOfMemory"、"Connection refused"
- 异常堆栈出现频率 > 5 次/分钟

告警内容：
服务：{service_name}
错误类型：{error_type}
出现次数：{count}
最近错误：
{last_error_message}
```

**方式四：网站可用性监控**

```
配置网站监控：

监控站点：
1. 主站
   - URL：https://example.com
   - 检查频率：每分钟
   - 超时时间：30 秒
   - 预期状态码：200
   - SSL 证书：提前 7 天告警

2. API 端点
   - URL：https://api.example.com/v1/health
   - 检查频率：每 30 秒
   - 响应时间告警：> 2 秒

告警策略：
- 单次失败：记录日志
- 连续 3 次失败：发送告警
- 恢复后：发送恢复通知
```

**方式五：自定义指标监控**

```
创建自定义监控：

指标名称：订单成功率
数据源：数据库查询
计算公式：成功订单数 / 总订单数 * 100%
检查频率：每 5 分钟

告警规则：
- 成功率 < 95%：警告
- 成功率 < 90%：严重
- 成功率 < 80%：紧急

通知策略：
- 警告：发送到值班群
- 严重：@值班人员
- 紧急：电话通知
```

**方式六：聚合告警**

```
配置告警聚合：

聚合窗口：5 分钟
相同告警：合并为一条
告警静默：
- 同一告警 30 分钟内只通知一次
- 维护窗口：凌晨 02:00-06:00 不告警

告警升级：
- 15 分钟未处理：升级到组长
- 30 分钟未处理：升级到经理
- 60 分钟未处理：电话通知
```

**OpenClaw 会自动：**
1. 按规则检查监控指标
2. 判断是否触发告警
3. 发送告警通知
4. 记录告警历史
5. 处理告警恢复

### 3. 效果展示

**告警通知：**
```
🚨 系统告警 - 严重

服务器：prod-web-01
指标：CPU 使用率
当前值：92%
阈值：80%
持续时间：8 分钟
时间：2024-03-24 14:23:00

建议操作：
1. 检查运行中的进程
2. 查看系统日志
3. 考虑扩容或重启服务

查看详情：https://monitor.example.com/alert/123
```

**恢复通知：**
```
✅ 告警已恢复

服务器：prod-web-01
指标：CPU 使用率
恢复值：65%
持续告警时长：23 分钟
恢复时间：2024-03-24 14:46:00

告警历史：
- 14:23 触发告警（92%）
- 14:35 达到峰值（98%）
- 14:46 恢复正常（65%）
```

**监控仪表盘：**
```
📊 系统监控状态

服务器健康：✅ 正常
- CPU：65%（阈值 80%）
- 内存：72%（阈值 85%）
- 磁盘：78%（阈值 90%）

服务状态：
✅ API 服务 - 响应时间 120ms
✅ 数据库 - 连接正常
✅ Redis - PONG
✅ 主站 - SSL 有效

今日告警：
- 总计：3 次
- 已恢复：3 次
- 进行中：0 次
```

| 依赖项 | 说明 | 获取方式 |
