# 定时备份

## 干什么
让 OpenClaw 自动执行定时备份任务，支持文件备份、数据库备份、配置备份等。

## 怎么干

### 1. 准备工作
- 需要 OpenClaw 版本：>= 1.0.0
- 需要的配置：备份目标路径、备份源路径
- 需要的权限：文件读写权限

### 2. 执行步骤

**方式一：设置定时文件备份**

```
创建定时备份任务：

任务名称：项目文档备份
备份源：~/Documents/projects
备份目标：/backup/projects
执行时间：每天 02:00
保留策略：保留最近 7 天的备份

备份内容：
- 所有 .md 文档
- 配置文件
- 忽略 node_modules 等临时文件
```

**方式二：数据库定时备份**

```
设置数据库自动备份：

数据库类型：PostgreSQL
数据库地址：localhost:5432
数据库名：production_db
备份目标：/backup/database
执行时间：每天 03:00
保留天数：30 天
压缩格式：gzip

备份前操作：
- 检查数据库连接
- 记录备份开始时间

备份后操作：
- 验证备份完整性
- 发送备份报告
- 清理过期备份
```

**方式三：增量备份**

```
配置增量备份：

备份类型：增量备份
基准：每周日全量备份
增量：周一至周六增量备份
备份源：/data/important
备份目标：/backup/incremental
保留策略：保留最近 4 周的备份链
```

**方式四：云存储备份**

```
设置云端备份：

备份源：/backup/local
备份目标：云存储（S3/OSS/COS）
执行时间：每天 04:00
同步方式：rsync
加密：启用 AES-256 加密
带宽限制：10 MB/s

通知：
- 成功：发送摘要到飞书
- 失败：发送告警到企业微信群
```

**方式五：多任务备份编排**

```
创建备份流程：

备份组：每日备份任务组
执行顺序：
1. 数据库备份（03:00）
2. 文件备份（03:30）
3. 配置备份（04:00）
4. 云端同步（04:30）

依赖关系：
- 文件备份依赖数据库备份完成
- 云端同步依赖所有本地备份完成

失败处理：
- 单个任务失败不影响其他任务
- 发送告警通知
- 记录失败详情
```

**OpenClaw 会自动：**
1. 按计划执行备份任务
2. 验证备份完整性
3. 清理过期备份
4. 发送备份报告
5. 处理异常和重试

### 3. 效果展示

**备份执行报告：**
```
✅ 备份任务完成

任务：项目文档备份
执行时间：2024-03-24 02:00:00
耗时：3 分 25 秒

备份详情：
- 文件数量：1,234 个
- 总大小：256 MB
- 压缩后：89 MB
- 备份位置：/backup/projects/2024-03-24/

保留策略：
- 当前保留：7 个备份
- 最早备份：2024-03-18
- 已清理：0 个过期备份

下次执行：2024-03-25 02:00:00
```

**备份失败告警：**
```
⚠️ 备份任务失败

任务：数据库备份
执行时间：2024-03-24 03:00:00
失败原因：数据库连接超时

错误详情：
Error: Connection to localhost:5432 timed out after 30000ms
最后成功连接：2024-03-23 03:00:00

建议操作：
1. 检查数据库服务状态
2. 检查网络连接
3. 手动重试备份

重试次数：3 次（均失败）
```

## 需要准备什么

| 依赖项 | 说明 | 获取方式 |
|--------|------|----------|
| OpenClaw | >= 1.0.0 | 官方安装 |
| 存储空间 | 备份存储 | 本地或云端 |
| 可选：云存储 | S3/OSS/COS | 云服务商 |
