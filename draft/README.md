# 草稿与灵感

这个目录存放**待补充真实来源**的案例草稿。

## 为什么存在这些草稿

这些案例目前缺少：
- 真实开源项目链接
- 真实技能文档引用
- 可验证的命令或配置

它们是"怎么干"的思路，而不是已经验证的真实案例。

## 如何转化成正式案例

1. 找到对应的开源项目 / 技能文档 / 真实命令
2. 在文件头部补充：
   ```md
   ## 真实来源
   - 项目：xxx
   - GitHub：https://github.com/xxx
   ```
3. 验证命令或配置可用
4. 移动到对应的正式目录
5. 更新 README
