# Notion 笔记操作

## 干什么
让 OpenClaw 自动操作 Notion，包括创建笔记、更新内容、管理数据库、搜索笔记等。

## 怎么干

### 1. 准备工作
- 需要 OpenClaw 版本：>= 1.0.0
- 需要的配置：Notion API Token
- 需要的权限：访问 Notion workspace

### 2. 执行步骤

**方式一：创建笔记**

```
在 Notion 创建一篇新笔记：

标题：项目周报 - 2024-03-24
内容：
## 本周完成
- 完成用户登录功能
- 修复 3 个 bug
- 性能优化

## 下周计划
- 开发支付模块
- 编写测试用例

保存到：工作/项目文档
```

**方式二：更新笔记**

```
更新 Notion 中的笔记：

找到笔记：API 设计文档
追加内容：
## 更新日志
- 2024-03-24：新增用户认证接口
- 接口地址：/api/auth
```

**方式三：管理数据库**

```
在 Notion 任务数据库添加任务：

数据库名：任务管理
添加记录：
- 任务名称：实现搜索功能
- 状态：进行中
- 优先级：高
- 负责人：开发团队
- 截止日期：2024-03-30
```

**方式四：搜索和查询**

```
搜索 Notion 笔记：

关键词：API 文档
范围：整个 workspace
筛选条件：
- 创建时间：最近 30 天
- 类型：文档

返回匹配的笔记列表。
```

**方式五：批量操作**

```
批量更新 Notion 数据库：

数据库名：项目跟踪
筛选条件：状态 = "待处理"
操作：将优先级设为"高"
```

**OpenClaw 会自动：**
1. 连接 Notion API
2. 执行笔记操作
3. 返回操作结果
4. 处理错误和重试

### 3. 效果展示

**创建笔记成功：**
```
✅ Notion 笔记已创建

笔记链接：https://notion.so/xxx
标题：项目周报 - 2024-03-24
位置：工作/项目文档
字数：120 字

已自动添加标签：#周报 #项目
```

**数据库更新：**
```
✅ 任务已添加到 Notion 数据库

任务详情：
- 名称：实现搜索功能
- 状态：进行中
- 优先级：高
- ID：task-20240324-001

数据库当前统计：
- 总任务：45
- 进行中：12
- 已完成：28
- 待处理：5
```

## 需要准备什么

| 依赖项 | 说明 | 获取方式 |
|--------|------|----------|
| OpenClaw | >= 1.0.0 | 官方安装 |
| Notion API Token | 访问权限 | Notion 设置中获取 |
| Notion workspace | 笔记空间 | notion.so |
